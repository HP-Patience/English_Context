# ADR-0002: 选词搜索复用组件

## 状态

已实现

## 日期

2026-06-21

## 背景

用户在背诵和复习单词时，看到句子中的不认识的词，需要手动复制、切换到搜索页、粘贴、搜索，链路过长。原有学习/复习页面中，句子以纯文本展示，没有任何交互相应——点击或长按文本无反馈。

需要一个通用的机制：任何包含文本的页面，用户选中单词后出现搜索入口，点击直接跳转搜索页并自动填入选中词。此机制应做成可复用组件，后续词库列表、单词详情、收藏夹等页面都能直接使用。

## 决策

### 方案：SelectionSearch wrapper 组件

不修改每个页面的数据流或路由逻辑，新建一个纯客户端 wrapper 组件，利用 DOM Selection API 监听用户文本选择，配合浮动按钮实现"选词→搜索"交互。

### 关键设计

#### 1. 组件接口

```
src/components/SelectionSearch.tsx
```

props 仅 `children` + `className`，与普通容器组件无异。任何已有文本段落直接包裹即可启用选词搜索。

#### 2. 选词检测

使用 `document.addEventListener('selectionchange', ...)` 监听全局选区变更。通过 `Range.getBoundingClientRect()` 获取选中区域视口坐标，计算浮动按钮定位（居中于选区上方）。

#### 3. 范围限制

通过 `containerRef.current?.contains(range.commonAncestorContainer)` 判断选区是否在组件内部，避免误触发其他区域的选词。

#### 4. 浮动按钮

按钮 `position: fixed`，根据选区坐标定位，z-50 确保浮于所有内容之上。点击后 `router.push('/search?q=...')` 跳转搜索页。

#### 5. 收起逻辑

以下情况隐藏按钮：
- 选区变空
- 点击按钮外部区域
- 滚动页面
- 按 Escape 键

#### 6. 可复用性

组件不依赖任何特定页面数据或 context，后续其他页面只需 `<SelectionSearch><p>some text</p></SelectionSearch>` 即可启用。

### 未选择的方案

- **右键菜单**：移动端无右键事件，且与浏览器默认菜单冲突，体验不一致。
- **单词高亮点击**：需预先知道句子中哪些是可点击词，对 AI 生成的动态句子不适用；且用户可能想搜索句子中任意的词，不限于当前学习的词。
- **长按识别**：需区分长按与滚动/滑动，实现复杂且各平台行为不一致。浏览器原生文本选择是用户最熟悉的交互模式。

### 影响

#### 正面
- 用户在任意页面均可快速查词，减少操作步骤
- 组件零外部依赖，纯 DOM API + React，轻量且与框架无关
- 未来新增页面只需引入包裹，无需修改组件本身

#### 负面
- 对不支持 `window.getSelection()` 的环境（TV、部分旧浏览器）不生效，影响可忽略
- 浮动按钮在选中跨多行文本时定位在最上方行，位置略偏（可用性可接受）
- 按钮样式硬编码在组件内，后续如需主题定制需抽取
